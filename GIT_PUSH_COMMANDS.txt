=============================================================================
GIT COMMANDS TO PUSH LOCATION FEATURE TO REPOSITORY
=============================================================================

Repository: https://github.com/pranavsoftware/NutriVision-Pro---AI-Powered-Food-Analyzer.git

-----------------------------------------------------------------------------
STEP 1: Initialize Git (if not already done)
-----------------------------------------------------------------------------

git init
git remote add origin https://github.com/pranavsoftware/NutriVision-Pro---AI-Powered-Food-Analyzer.git

-----------------------------------------------------------------------------
STEP 2: Check Current Status
-----------------------------------------------------------------------------

git status

This will show all modified and new files:
- Modified: app.py, templates/dashboard.html, static/css/analyzer.css, README.md
- New: LOCATION_FEATURE.md, IMPLEMENTATION_SUMMARY.md, test_location_feature.py

-----------------------------------------------------------------------------
STEP 3: Stage All Changes
-----------------------------------------------------------------------------

git add .

Or stage files individually:

git add app.py
git add templates/dashboard.html
git add static/css/analyzer.css
git add README.md
git add LOCATION_FEATURE.md
git add IMPLEMENTATION_SUMMARY.md
git add test_location_feature.py

-----------------------------------------------------------------------------
STEP 4: Commit Changes
-----------------------------------------------------------------------------

git commit -m "feat: Add location-based nearby places discovery feature

- Implement geolocation API integration for automatic location access
- Add get_nearby_places() function using Gemini API for place suggestions
- Update /upload endpoint to accept latitude and longitude parameters
- Display 4-5 nearby shops, restaurants, and hotels in results
- Add location data storage in MongoDB
- Implement privacy-first approach with graceful degradation
- Add comprehensive documentation and test suite
- Update README with new feature details

Version: 1.1.0"

-----------------------------------------------------------------------------
STEP 5: Pull Latest Changes (if repository already exists)
-----------------------------------------------------------------------------

git pull origin main --rebase

Or if using 'master' branch:

git pull origin master --rebase

If there are conflicts, resolve them and continue:

git add .
git rebase --continue

-----------------------------------------------------------------------------
STEP 6: Push to GitHub
-----------------------------------------------------------------------------

# For main branch:
git push origin main

# For master branch:
git push origin master

# If this is the first push:
git push -u origin main

# If you need to force push (use carefully):
git push origin main --force

-----------------------------------------------------------------------------
STEP 7: Verify Push
-----------------------------------------------------------------------------

Visit your repository:
https://github.com/pranavsoftware/NutriVision-Pro---AI-Powered-Food-Analyzer

Check:
- All files are uploaded
- Commit message is visible
- README.md displays correctly with new features

=============================================================================
TROUBLESHOOTING
=============================================================================

Issue: "fatal: not a git repository"
Solution: Run 'git init' first

Issue: "rejected - non-fast-forward"
Solution: Run 'git pull origin main --rebase' then push again

Issue: "remote origin already exists"
Solution: This is fine, skip the 'git remote add' command

Issue: Authentication failed
Solution: Use GitHub Personal Access Token instead of password
         Generate at: https://github.com/settings/tokens

Issue: Branch name mismatch (main vs master)
Solution: Check your default branch:
         git branch -a
         Use the correct branch name in commands
